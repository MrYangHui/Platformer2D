# 芬妮整帧动画抖动修复设计

日期：2026-07-18
状态：用户已批准方案方向，等待书面复核

## 1. 问题与证据

当前整帧运行时只切换 `SpriteRenderer.sprite` 与 `flipX`，不会逐帧修改 Transform。因此抖动不来自整帧架构本身，而来自最终图集中角色视觉锚点不连续。

现有 v006 图集的离线测量结果如下：

- Idle 骨盆横坐标为 363–394 px，骨盆高度为 462–488 px；
- Run 骨盆横坐标为 377–411 px，骨盆高度为 263–352 px；
- 在当前 PPU 与相机缩放下，Run 最坏会造成约 4 个屏幕像素的横向跳变和约 10 个屏幕像素的纵向跳变；
- `normalize_character_frames.py` 对 Grounded 帧同时使用承重脚对齐 X/Y。跑步左右脚交替后，承重脚 X 不是稳定的身体参考点，导致全身横移；
- Idle/Run 源图由独立姿势组成，虽然头部—骨盆距离已归一化，但没有受控的骨盆轨迹与成对周期约束。

此前测试只覆盖画布、Pivot、比例、脚底线和运行时 Transform，不覆盖动画序列的视觉连续性，这是本次需要补齐的验收缺口。

## 2. 方案比较与决定

### 方案 A：保留整帧架构，重建连续帧生产管线（采用）

重新制作 Idle/Run 源素材，Grounded 横向使用骨盆对齐、纵向使用脚底线，并对最终骨盆轨迹实施自动预算。优点是保持完整人体轮廓、运行时简单、后续角色可复用同一合同；代价是每个角色都必须提供经过连续性验收的完整动画帧。

### 方案 B：恢复 Unity 2D 骨骼动画

通过骨骼插值获得连续动作。优点是运动天然平滑；缺点是当前生成式分件缺少可靠蒙皮与关节遮挡，已经出现木偶拼接问题，且新角色需要重新分件、权重与绑定。

### 方案 C：运行时逐帧位移补偿

为每张 Sprite 保存偏移并在播放时修正。短期可以压住部分抖动，但会把错误素材隐藏进运行时数据，并重新引入逐帧 Transform 跳变，不利于后续角色维护，因此拒绝。

## 3. 素材与动作合同

### 3.1 Idle

- 首次修复阶段只播放一张经过确认的 Idle 基准帧，彻底消除静止换帧抖动；
- 后续若恢复多帧呼吸动画，必须由同一基准帧派生，只允许头发、胸腹或衣摆的小幅局部变化；
- 多帧 Idle 的骨盆 X/Y 漂移均不得超过 4 px。

### 3.2 Run

- 使用 8 帧完整循环：接触、下压、经过、腾空，以及对应的另一侧四相；
- 所有帧保持同一角色身份、服装、头部尺寸、躯干长度、背包尺寸与侧视角；
- 每个相位必须从相邻相位演进，禁止将八张独立跑姿直接拼成序列；
- 前后半周期必须成对：`Run_00` 对应 `Run_04`、`01` 对应 `05`、`02` 对应 `06`、`03` 对应 `07`；
- 骨盆纵向轨迹允许自然起伏，但总范围不得超过 32 px，相邻帧变化不得超过 18 px；
- 骨盆横向在最终单元格中的漂移不得超过 8 px，相邻帧变化不得超过 6 px。

### 3.3 尺度与接地

- 单元格仍为 768×1024，PPU 仍为 480，Sprite Pivot 仍为底边中心 `(0.5, 0)`；
- 一个角色只使用一次统一尺度计算，禁止按透明包围盒逐帧缩放；
- Grounded 帧的 X 轴对齐 `PelvisAnchor` 到单元格中心；
- Grounded 帧的 Y 轴对齐 `SoleAnchor` 到统一脚底线；
- 空中帧继续使用骨盆对齐，不修改本次已经被用户接受的跳跃姿势；
- 所有对齐烘焙进最终图集，运行时不新增逐帧位置或缩放字段。

## 4. 工具与数据流

`normalize_character_frames.py` 增加轴向分离对齐：Grounded 使用骨盆 X 与脚底 Y。归一化完成后，工具基于清单中的动作组执行序列连续性验证；任何帧超过预算时，必须在覆盖正式图集前失败并报告帧名、实际差值和允许值。

素材流程为：

1. 以当前芬妮完整角色图与已确认外观作为身份参考；
2. 生成或编辑一套连续的 Idle/Run 候选源图；
3. 人工标注 Sole、Pelvis、Head；
4. 归一化并执行比例、边界和抖动预算检查；
5. 输出接触表和真实游戏缩放的动画预览；
6. 只有预览通过才覆盖 Profile 引用的版本化图集。

## 5. 运行时行为

- `PlayerFramePresentation2D` 继续只选择 Sprite 和朝向；
- Idle 在本轮使用单帧，不启动无意义的计时循环；
- Run 仍随水平速度在有限范围内调整播放速度；
- 不修改 Rigidbody2D、碰撞体、GroundProbe、PlayerMotor、相机或关卡；
- 不使用每帧 Transform 校正掩盖素材问题。

## 6. 测试策略

### 6.1 先失败的回归测试

- 当前 v006 Run 骨盆纵向范围超过 32 px，测试必须先失败；
- 当前 Grounded 对齐将脚底 X 固定在中心，骨盆 X 漂移超标，测试必须先失败；
- 多帧 Idle 的骨盆漂移超标，测试必须先失败。

### 6.2 工具测试

- Grounded 输出的骨盆 X 对齐单元格中心；
- Grounded 输出的脚底 Y 对齐统一脚底线；
- 超过序列预算时拒绝输出，并指出具体动作与帧；
- 合格的对称 8 帧 Run 循环通过；
- 重复执行生成相同图集与清单结果。

### 6.3 Unity 回归

- Edit Mode 验证 Sprite 数量、Pivot、PPU、Profile 引用与单帧 Idle；
- Play Mode 验证静止一秒内 Sprite 不切换，移动时只切 Sprite/flipX；
- 完整 Edit Mode、Play Mode、Meta 审计与 Windows x64 Development Build 通过。

## 7. 人工验收

- 角色静止至少五秒完全稳定；
- 左右各连续跑动十秒，头部、躯干和背包不存在可感知的横向跳点；
- 跑步允许自然的轻微重心起伏，但不得出现整个人快速升降或忽大忽小；
- 接触脚不滑动，腾空相位不被强行压到地面；
- 停止、转向、起跑与落地切换不产生单帧闪跳；
- Play Mode 与 Windows Development Build 表现一致。

## 8. 回退与范围

新图集使用新版本文件名，v006 和旧骨骼素材继续保留。若新候选帧未通过抖动预算或人工预览，Player 继续引用当前已提交资源，不覆盖正式素材。

本次只修复芬妮 Idle/Run 与通用连续性校验，不增加攻击、刹车、转身或新角色，也不修改玩法参数。

## 9. 完成标准

- Idle 单帧稳定且五秒内无切帧；
- Run 骨盆横向总漂移不超过 8 px、相邻不超过 6 px；
- Run 骨盆纵向总范围不超过 32 px、相邻不超过 18 px；
- 运行时无逐帧位置/缩放补偿；
- 自动测试、真实缩放动画预览、Play Mode 与 Windows Build 全部通过；
- 用户确认移动与静止抖动已消失后，再将该素材合同用于后续角色。
