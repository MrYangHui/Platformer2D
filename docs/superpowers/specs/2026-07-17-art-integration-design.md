# 第一关贴图素材集成设计

日期：2026-07-17  
状态：用户已批准方案 A，等待书面复核

## 1. 目标与范围

把已经验收的第一关环境、平台和芬妮候选素材接入可玩切片，让 Play Mode 与 Windows 构建具有完整的冰封禁区画面，同时保持现有移动手感、碰撞判定、检查点、收集物和关卡路线不变。

本轮包含：

- Sky、Far、Mid、Near 四层背景与基础循环视差。
- 短、中、长平台预制体的正式贴图替换。
- 芬妮 Idle 四帧和 Run 八帧的临时整帧动画。
- 贴图导入、裁切、枢轴、排序层和运行时引用的自动化验证。

本轮不包含：

- 芬妮 2D 骨骼、Sprite Skin、权重和最终关节修图。
- Jump、Fall、Land、Brake 等正式动画资源。
- 新玩法、移动参数、碰撞体尺寸或关卡几何修改。
- 新背景变体、无缝重绘、粒子、后处理或动态天气。

## 2. 方案选择

采用已批准的渐进式混合集成方案：平台先使用正式候选贴图，背景使用轻量循环视差，芬妮使用整帧临时动画。该方案能立即验证完整画面，又不会把尚未人工整理的部件母版错误地当作正式骨骼资源。

未采用纯静态场景铺图，因为长关卡会明显暴露背景接缝且缺少景深。未采用立即制作骨骼动画，因为当前部件母版仍存在头发层级、复合手臂、裙摆重叠和关节余量需要人工清理的问题。

## 3. 平台贴图集成

`Platform_Short`、`Platform_Medium`、`Platform_Long` 三个预制体分别引用对应候选图。平台根节点、Layer、BoxCollider2D、单向平台设置和场景实例位置保持不变，只修改子节点 `Visual` 的 SpriteRenderer 表现。

导入与裁切规则：

- 使用候选图中已去背的平台主体区域，不把透明画布边距计入运行时 Sprite。
- 枢轴统一为中心，视觉中心继续与碰撞体中心对齐。
- 短平台裁出约 3:1 的代表性模块，中平台裁出约 6:1 的主体，长平台使用完整约 12:1 主体，降低非等比拉伸。
- SpriteRenderer 继续使用固定 `size`：短 `(3,1)`、中 `(6,1)`、长 `(12,1)`。
- 使用 Terrain 排序层；不从 Sprite 自动生成碰撞或物理形状。

验收时必须比较 SpriteRenderer 世界包围盒与 BoxCollider2D 世界包围盒，宽高误差不超过 `0.01` 世界单位。

## 4. 背景与视差架构

在关卡场景建立独立的 `EnvironmentVisuals` 根节点，包含：

- `Background_Sky`：覆盖相机视野的云层基底，基本跟随相机，保证任何关卡位置都无灰色露底。
- `Background_Far`：远景城市和封锁设施，缓慢视差。
- `Background_Mid`：建筑、道路残骸和隔离设施，中等视差。
- `Background_Near`：钢架与管线，较明显视差，降低颜色亮度和透明度以避免误判为玩法平台。

新增 `ParallaxLayer2D` 表现组件。每层由三个 SpriteRenderer 段组成，组件在 `LateUpdate` 中：

1. 使用显式序列化的相机 Transform；缺失时报告配置错误并禁用图层。
2. 根据相机位置和水平／垂直跟随系数移动图层根节点。
3. 根据单段世界宽度，把三个段循环排列到相机附近。
4. 保持 Z、排序层和段内局部 Y 不变，不参与任何物理运算。

视差由远到近逐渐增大，但所有系数都以可读性为先。Near 层不得遮挡芬妮、平台上缘、收集物或检查点。背景素材并非严格无缝图，因此段间允许少量透明重叠，并用层间雾色与低对比弱化接缝。

## 5. 芬妮临时动画表现

保留 Player 根节点、Rigidbody2D、CapsuleCollider2D、GroundProbe、CameraTarget 和 PlayerMotor2D 不变。只替换 `Visual` 子节点的 SpriteRenderer，并新增 `PlayerSpriteAnimator2D` 表现组件。

组件输入：

- `PlayerMotor2D.State`
- Rigidbody2D 的水平速度
- Idle 四帧与 Run 八帧 Sprite 数组

组件输出：

- 静止且接地时循环 Idle。
- 接地且水平移动时循环 Run；速度绝对值可轻微影响播放速率，但必须限制在稳定范围内。
- Rising 或 Falling 时暂时保持一个经过选择的稳定姿势，不伪造未生成的跳跃动画。
- 水平速度超过小阈值时更新面向方向；向左通过 SpriteRenderer `flipX` 或视觉根节点镜像实现，物理根节点不翻转。

角色整帧 Sprite 统一使用底部中心枢轴，并通过每张表独立 PPU 归一到约 `1.9` 世界单位的站立高度。Idle 和 Run 切换时脚底基线不得明显跳动，视觉轮廓可以略高于碰撞体，但不能改变主体碰撞。

该组件属于临时表现适配器。后续骨骼动画将替换内部 Sprite 选择逻辑，但继续消费 PlayerMotor2D 的运动状态，不要求改动玩家物理代码。

## 6. 错误处理与边界

- `ParallaxLayer2D` 缺少相机、段 Sprite 或有效段宽时，输出包含对象路径的明确错误并禁用自身。
- `PlayerSpriteAnimator2D` 缺少 SpriteRenderer、Rigidbody2D、PlayerMotor2D 或 Idle/Run 帧时，输出明确错误并禁用自身；玩家物理仍可运行。
- 所有运行时组件都使用显式序列化引用，不按对象名称查找玩法对象。
- 编辑器配置脚本只负责可重复地设置贴图导入、预制体和场景表现引用，不改写玩家移动配置或关卡几何。

## 7. 测试策略

### 7.1 EditMode

- 验证视差循环位置计算能把三段稳定覆盖相机附近区域。
- 验证动画帧索引在 Idle 四帧和 Run 八帧中正确循环。
- 验证速度为负时产生左向表现，接近零时保留最后面向方向。

### 7.2 PlayMode

- 验证三类平台 SpriteRenderer 使用正确候选 Sprite，且世界包围盒继续匹配碰撞体。
- 验证关卡存在 Sky/Far/Mid/Near 四层、正确排序层、每个循环层具有三个段且无 Collider2D。
- 验证 Player 使用芬妮 Sprite、Idle 四帧与 Run 八帧引用完整，并且视觉变化不改变 Rigidbody2D、CapsuleCollider2D 或移动配置。
- 继续运行既有关卡结构、路线可达性、收集、复活和相机测试。

### 7.3 构建与人工验收

- 完整 EditMode 和 PlayMode 零失败。
- Windows x64 Development Build 成功。
- 人工检查 16:9 Play Mode：无背景露底、明显接缝闪烁、平台碰撞错位、角色脚底跳动或前景遮挡玩法目标。

## 8. 完成标准

- 从第一关起点到终点全程显示冰封禁区背景与平台贴图。
- 平台视觉和真实判定保持一致。
- 芬妮能正确显示、Idle、Run 和左右转向，跳跃期间保持稳定临时姿势。
- 玩家移动、碰撞、路线、存档点与收集流程无回归。
- 素材和来源记录继续位于既定目录，运行时只引用 `Assets/Game/Art/` 下的候选资源。
- 已知的骨骼清理、正式跳跃动作和背景无缝重绘继续作为后续独立制作任务，不在本轮伪装完成。
