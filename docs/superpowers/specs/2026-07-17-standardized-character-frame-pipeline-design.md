# 标准化整帧角色表现管线设计

日期：2026-07-17  
状态：用户已批准方案方向，等待书面复核

## 1. 目标

用“标准化整帧动画为主、非变形挂件为辅”的角色表现管线替换当前芬妮刚性分件木偶。新方案必须同时满足：

- 在实际游戏缩放下保持完整、连续的人体与服装轮廓，不暴露肩、肘、髋、膝或踝部切口；
- 消除不同动画帧造成的整体放大、缩小、裁切框漂移和脚底跳动；
- 让 Idle、Run、Rising、Apex、Falling 的姿势切换自然，并保留已经确认的脚底压入平台效果；
- 新角色只需按统一素材模板提供动作帧和一个表现配置，不重写状态机或玩家物理代码；
- 不修改 `Rigidbody2D`、`CapsuleCollider2D`、`GroundProbe2D`、`PlayerMotor2D`、移动参数、相机或关卡几何。

## 2. 根因与架构决策

当前刚性分件没有失败在 Unity 插值本身，而是失败在输入素材。`FennyGolden_RigParts_v005.png` 中的肢体由独立生成的硬切零件组成，存在空心肩窝、截断关节、比例差异和不足的遮挡余量。刚性旋转会暴露这些截面；排序和扩大重叠只能改善少量姿势，无法同时保证 Idle、Run 和 Airborne 的完整轮廓。

项目最初的角色规范已经要求最终骨骼素材必须人工整理关节覆盖区，不能把生成素材直接切开后绑定。当前没有持续的人工分件、蒙皮和逐姿势修形条件，因此不再把刚性分件作为游戏内主体角色方案。

采用整帧主体的原因：

- 每一帧都是完整构图，人体、服装、头发和背部装备的遮挡关系由原图保证；
- 生成式素材更擅长输出完整姿势，而不是严格匹配的工程分件；
- 新角色可以复用同一导入、归一化、状态选择和测试管线；
- 运行时只需一个主体 `SpriteRenderer`，逻辑与性能成本可预测。

刚性骨骼资产继续保留在仓库中作为实验记录和回退参考，但从 Player 预制体移除，不在运行时加载。

## 3. 素材标准

### 3.1 动作集合

每个可玩角色的首版表现配置至少包含：

- Idle：至少 2 帧，目标 4 帧；
- Run：至少 8 帧，按接触、下压、经过、腾空四阶段组织左右腿循环；
- Rising：至少 1 帧；
- Apex：至少 1 帧；
- Falling：至少 1 帧。

Landing、Brake、Turn、Attack 和受击属于后续可选状态，不作为本次芬妮迁移的前置条件。缺少可选状态时，适配器使用明确的核心状态回退，不动态拼接肢体。

### 3.2 固定画布与角色尺度

生产帧使用统一透明单元格，目标上限为 `768 x 1024` 像素。宽度为伸展跑姿保留透明空间，高度、PPU 与角色参考尺度保持统一；一个角色的所有动作使用同一单元格尺寸、同一 PPU 和同一个角色参考尺度。

尺度只根据该角色的站立参考帧确定一次，随后应用到全部动作帧。禁止根据每一帧的透明包围盒分别缩放，因为跑步下蹲、抬腿和空中收腿本来就会改变包围盒高度，逐帧归一化会重新制造呼吸式缩放。

芬妮的名义站立视觉高度保持约 `1.9` 世界单位。未来角色可以拥有不同的名义视觉高度，但同一角色内部只能有一个尺度值；碰撞体差异属于独立玩法设计，不由表现资源自动修改。

### 3.3 锚点规则

离线素材清单为每帧记录以下像素锚点：

- `SoleAnchor`：接地状态中承重脚的接触点；
- `PelvisAnchor`：骨盆中心，用于跨状态和空中姿势对齐；
- `HeadAnchor`：头部参考点，只用于比例质量检查，不作为运行时位移来源。

Grounded 帧将 `SoleAnchor` 对齐到统一脚底线；Rising、Apex、Falling 帧按 `PelvisAnchor` 对齐到共同角色尺度。运行时不保存逐帧位置补丁，所有帧在生成最终图集前已经被烘焙到正确位置。

最终图集中的所有 Sprite 使用相同的固定 Pivot：单元格底边中心 `(0.5, 0)`。统一脚底线位于单元格内的固定像素高度，角色级视觉根偏移负责把该脚底线对齐到平台；Pivot 和视觉根都不会随帧变化。

### 3.4 比例质量门槛

归一化工具比较各帧的头部—骨盆距离、骨盆—脚底参考长度和躯干宽度。与站立参考相比，非姿势原因造成的比例偏差超过 3% 时拒绝生成最终图集并报告帧名。自动检查无法判断的衣物、手指、头发和遮挡问题必须通过接触表人工复核。

该检查用于拒绝角色在不同帧中真正“长高、变宽或换脸”的素材；它不会抹平跑步应有的骨盆起伏和肢体伸展。

## 4. 编辑器与资源管线

新增一个确定性的角色帧归一化工具，输入为版本化源图、帧清单和锚点数据，输出为版本化透明图集。工具负责：

1. 验证帧数量、透明通道、画布边界和唯一名称；
2. 按角色级统一尺度放置所有帧；
3. 按状态对应的锚点规则烘焙位置；
4. 生成固定单元格图集及 Unity Sprite 切片数据；
5. 生成 Idle、Run 和空中状态接触表供人工视觉复核；
6. 重复执行时保持图集、Sprite 名称和配置引用稳定。

图集使用透明 Sprite、统一 PPU、Full Rect、无 Mipmap 和适合 PC 目标的高质量透明纹理压缩。运行时不引用生成源图，只引用 `Assets/Game/Art/Characters/` 下验收后的版本化图集。

芬妮迁移优先尝试现有 v003 Idle/Run 和 v004 Airborne 完整姿势。任何未通过比例门槛或接触表复核的帧单独重新制作，不因少数坏帧重做已验收素材。

## 5. 运行时架构

### 5.1 `CharacterPresentationProfile`

每个角色拥有一个 ScriptableObject 表现配置，包含：

- Idle、Run 帧数组；
- Rising、Apex、Falling Sprite；
- Idle 与 Run 播放速度；
- 移动阈值和 Run 参考速度；
- 一个角色级视觉根偏移和统一显示尺度；
- 缺失可选状态时使用的回退 Sprite。

配置不包含逐帧位置、逐帧缩放或玩家物理数据。芬妮和未来新角色使用相同配置类型。

### 5.2 `PlayerFramePresentation2D`

Player 使用一个通用整帧表现适配器。它只读取：

- `PlayerMotor2D.State`；
- `Rigidbody2D.linearVelocity`；
- 当前 `CharacterPresentationProfile`。

输出只作用于主体 `SpriteRenderer` 和固定视觉根：

- Grounded 且低速：循环 Idle；
- Grounded 且移动：循环 Run，播放速度可在有限范围内随水平速度变化；
- Rising：显示 Rising；
- 垂直速度接近零：显示 Apex；
- Falling：显示 Falling；
- 水平速度超过阈值时更新朝向，停止时保持最后朝向。

视觉根的位置和尺度只在加载角色配置时设置一次。换帧不得修改 Transform 位置或缩放，朝向使用 `SpriteRenderer.flipX`，避免负缩放传播到挂件。

### 5.3 局部挂件

主体轮廓中的头发、衣服、四肢和背包默认全部保留在整帧 Sprite 内。局部挂件仅用于不会破坏人体轮廓的内容，例如枪口闪光、武器发光、交互标识和短时特效。

挂件使用角色级命名锚点，不参与主体动作拼装。没有对应挂件资源时不影响角色显示。

## 6. 新角色接入流程

新角色加入时执行统一流程：

1. 提供固定侧视身份参考和服装参考；
2. 按标准动作集合制作完整姿势；
3. 标记 Sole、Pelvis、Head 锚点；
4. 运行归一化和比例检查，修复未通过的单帧；
5. 检查自动生成的接触表；
6. 创建一个 `CharacterPresentationProfile` 并引用最终图集；
7. 在角色选择或 Player 配置中替换 Profile。

状态选择逻辑、物理对象和关卡不为新角色复制。只要动作集合满足合同，新角色不需要新的 MonoBehaviour 或专用 Animator Controller。

## 7. 错误隔离与回退

- 缺少 Profile 或主体 SpriteRenderer 时，适配器输出包含对象路径的错误并只禁用自身，不影响玩家物理和关卡加载；
- 当前状态缺少 Sprite 时，使用配置中的回退 Sprite，并只记录一次明确警告；
- 帧数、锚点或比例检查失败时，编辑器工具停止生成，不覆盖上一个已验收图集；
- 归一化输出使用新版本文件名，旧的 v003/v004 和刚性骨骼资源在用户验收前不删除；
- Player 迁移失败时，可以重新挂回上一个已提交的表现预制体，而无需回滚关卡或运动系统。

## 8. 测试与视觉验收

### 8.1 EditMode

- 图集中的所有 Sprite 具有相同矩形尺寸、PPU 和 Pivot 规则；
- 一个角色的动作帧只使用一个尺度，最终配置不存在逐帧偏移或逐帧缩放字段；
- 必需状态完整，空数组、重复名称、越界锚点和比例超差会被拒绝；
- 归一化工具重复运行不会改变已验收资源引用；
- Player 预制体引用通用整帧适配器，不再引用运行时刚性骨骼预制体。

### 8.2 PlayMode

- Idle、Run、Rising、Apex、Falling 根据运动状态正确切换；
- 左右移动只改变 Sprite 朝向，不改变角色根或视觉根尺度；
- 停止移动后保留最后朝向；
- `Rigidbody2D`、`CapsuleCollider2D`、GroundProbe 和移动配置与迁移前一致；
- Bootstrap、关卡、UI、检查点、收集和复活回归测试继续通过。

### 8.3 人工验收

- Idle 中角色不悬空、不穿入平台，脚底维持当前已确认的轻微重叠；
- 完整左右 Run 循环中头部、躯干和装备不存在可感知的整体放大缩小；
- 换帧时骨盆轨迹连续，接触脚没有无原因的水平滑动或垂直跳点；
- 所有姿势的人体和衣物轮廓完整，不出现透明关节缝或木偶零件感；
- Rising、Apex、Falling 的腿部姿势符合重心变化，落地切回 Grounded 时无根节点跳变；
- 在实际游戏相机缩放和 Windows Development Build 中复核，而不是只查看高分辨率源图。

## 9. 性能与范围

运行时主体只有一个 SpriteRenderer，不执行骨骼蒙皮、逐帧 Transform 校正或每帧资源查找。图集按 PC 目标启用透明纹理压缩；首版每角色约 13 至 16 帧，资源成本在当前项目规模内可接受。

本轮只迁移芬妮并建立可复用管线，不制作第二个新角色，不实现角色选择 UI，不删除旧素材，也不修改攻击、受击和剧情表现。第二个角色将在芬妮验收后作为独立任务验证接入成本。

## 10. 完成标准

- 芬妮在 Idle、Run 和完整空中阶段保持原效果图的整体角色感，不再显示为分件木偶；
- 已解决的脚底接触效果继续保留；
- 不再出现整体缩放脉冲、裁切漂移或逐帧位置补丁；
- 新角色接入只需最终动作图集、锚点清单和一个表现 Profile；
- 完整 EditMode、PlayMode、资源 Meta 审计和 Windows x64 Development Build 全部通过；
- 用户在 Play Mode 中确认跑步与跳跃动作自然后，刚性骨骼才从 Player 运行路径中正式退役。
