# 芬妮角色表现修复设计

## 1. 目标

修复当前第一关中芬妮的三个明显表现问题：站立时靴底与平台之间存在视觉间隙、跑动换帧时角色明显抖动且节奏偏慢、空中姿势的双腿动作不够自然。所有改动仅限表现层，不改变 Rigidbody2D、CapsuleCollider2D、GroundProbe、PlayerMotor2D 或 PlayerMovementConfig。

## 2. 根因

角色碰撞体高度为 1.8，视觉根节点当前位于本地 Y = -0.9，因此物理底面本身正确落在平台上。截图中的间隙来自贴图可见靴底、独立切片边界和视觉根节点之间的组合误差，而不是碰撞悬空。

Run 八帧使用独立紧边界切片，宽度为 189–336 像素、高度为 382–436 像素，但全部以各自裁切框的底部中心作为 Pivot。人物躯干在每个裁切框中的相对位置不同，换帧时因此发生水平和垂直跳动。当前 12 FPS 会增强离散感，但直接提速并不能消除锚点误差。

空中状态当前复用 Run 第 4 帧，没有专门的跳跃构图，因此双腿同时大幅倾斜，无法满足期望姿势。

## 3. 采用方案

采用“视觉基线 + 每帧校正 + 专用空中帧”的渐进方案，不在本轮启动 2D 骨骼绑定，也不重新生成整套跑步动画。

### 3.1 贴地基线

Player 预制体的 `Visual` 基准位置从 `(0, -0.9, 0)` 调整为 `(0, -0.95, 0)`。该 0.05 世界单位修正只移动 SpriteRenderer，不改变玩家根节点或任何物理组件。Idle 状态靴底与平台上缘之间的可见间隙应不超过一个最终显示像素，并且不得出现明显穿入平台的情况。

### 3.2 跑步稳定与节奏

`PlayerSpriteAnimator2D` 增加序列化的视觉根节点、基准位置和八个 Run 帧校正偏移。每个偏移以骨盆和躯干为共同根锚点进行校准，抵消独立紧边界切片造成的位置漂移，同时保留跑步动作本身合理的轻微起伏。

动画状态切换时必须显式写入对应视觉偏移，Idle 使用零校正，Run 使用当前帧校正，空中使用独立空中校正。停止移动和落地后必须恢复正确状态偏移，不能继承上一帧 Run 校正。

Run 播放速度从 12 FPS 提高到 16 FPS；仍使用现有八帧，单次循环为 0.5 秒。先消除锚点抖动再提速，避免把位置误差放大为更快的抖动。若人工验收仍认为过渡不足，后续再作为独立任务补中间帧。

### 3.3 专用空中帧

生成一张新的芬妮侧视空中 Sprite，保持现有角色的脸、金色卷发双马尾、机械发饰、橙红黑白服装、绿色点缀、武器背包和左右腿不对称设计。

姿势要求：红色长袜腿保持大幅屈膝并向前上方收起；裸腿自然向下伸展，膝盖只做轻微弯曲，脚尖自然下垂；躯干保持略微前倾但不蜷缩。画面不得增加攻击特效、武器动作、阴影或地面。

新素材使用版本化文件名，不覆盖 Run 素材。导入后使用单 Sprite、底部中心 Pivot，并按约 1.9 世界单位角色高度配置 PPU。运行时 `airborneFrame` 改为引用该专用素材，不再引用任何 `FennyGolden_RunPoses` 帧。

## 4. 组件与数据流

`PlayerSpriteAnimator2D` 继续从 `Rigidbody2D.linearVelocity.x` 和 `PlayerMotor2D.State` 读取表现状态：

1. Grounded 且水平速度低于阈值：播放 Idle，并使用基准位置。
2. Grounded 且水平速度高于阈值：以 16 FPS 播放 Run，并应用当前帧校正。
3. Rising 或 Falling：显示专用空中帧，并应用空中校正。
4. 朝向仍由水平速度更新；速度归零时保持最后朝向。

校正只作用于 `Visual` 子节点。玩家根节点、相机目标、地面探针和碰撞体保持原位置。

## 5. 错误隔离

适配器启动时验证 Visual、SpriteRenderer、Rigidbody2D、PlayerMotor2D、Idle/Run 帧、专用空中帧和 Run 校正数组。Run 校正数量必须与 Run 帧数量一致。配置无效时只禁用表现适配器并输出一条明确错误，不影响玩家物理对象和关卡加载。

## 6. 验证

自动验证包括：

- EditMode：专用空中 Sprite 存在、底部中心 Pivot、世界高度与 Idle/Run 可比，Run 校正数量为八。
- PlayMode：Idle、Run、左右朝向、落地恢复和空中专用帧切换正确。
- 物理回归：重力、碰撞体尺寸、GroundProbe 和移动配置没有变化。
- 全量 EditMode、PlayMode、资源 Meta 配对审计和 Windows x64 Development Build。

人工验收包括：

- Idle 靴底贴住平台上缘，没有明显悬空或穿模。
- 连续向左、向右跑动时躯干不发生突兀的横向或纵向跳动。
- 16 FPS 的八帧循环比当前版本更连贯，且没有明显加速抖动。
- 空中姿势保持红色长袜腿大幅弯曲，裸腿自然下垂并轻微弯曲。

## 7. 非目标

- 本轮不创建 12 帧或更多帧的新版 Run 动画。
- 本轮不接入 2D Animation、Sprite Skin、骨骼、权重或 IK。
- 本轮不修改移动速度、跳跃高度、重力、碰撞判定、关卡路线或相机手感。
